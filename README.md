# A02
How to Set Up VS Code and GitHub:
1. Prereqs: Download Vs Code (https://code.visualstudio.com/download) and GitHub(https://git-scm.com/downloads)

2. Extensions Needed: Open VS Code, click extensions icon, the squares on the left, install GitHub Pull & Request extension (https://marketplace.visualstudio.com/items?itemName=GitHub.vscode-pull-request-github)

3. Account: Create a GitHub account on GitHub.com (if you don't already have an account)

4. Connection: In VS Code, look for the accounts icon, person circle at the bottom left. Click the 'turn on settings sync' button and sign in to your GitHub account

5. Repo: To start working on a project, press the 'f1' button in VS Code, type 'Git: Clone', and paste your repository URL.

6. Commit: Make all the changes needed to the file and click the source control icon, the branch icon on the left. Type a small message in the box and hit commit. Click sync changes to push your work to GitHub. 
   

Glossary:
* **Branch**: In Git, a branch is basically a separate workspace where you can make changes and try new things out without affecting your main project.
* **Clone**: A copy of your repository.
* **Commit**: A change to your file.
* **Fetch**: downloads commits, files, and refs from a remote repository into your local repo.
* **Git**: GitHub makes tools that use Git, an open-source control system.
* **GitHub**: The largest host of source code in the world. Git and GitHub are NOT the same. 
* **Merge**: Combines changes from your different branches.
* **Merge Conflict**: This occurs when competing changes are made to the same line of a file.
* **Push**: Sends out your changes to a remote repository.
* **Pull**: Returns the latest changes from a remote repository.
* **Remote**: This is a bookmark for another Git repository.
* **Repository**: This is a folder where Git tracks your project and its history.
